VIBE运行尝试

发表于 2020-03-25 | 分类于 CVPR2020开源项目

VIBE运行尝试

VIBE 全名 Video Inference for Human Body Pose and Shape Estimation[CVPR-2020] 官方开源代码链接:github项目地址。论文链接：arXiv。VIBE结合了人体姿态估计与人体形状估计，对于输入视频的每一帧，VIBE都相应地预测出SMPL人体模型的参数

运行环境

  • ubuntu 18.04
  • PY37

运行准备

首先尝试在win10下运行

官方在github上给出了linux下的sh脚本，win下自己按照脚本内容自己配置也很简单，这里就不重复写了，我在安装pytorch时遇到了以下问题

ERROR: Could not find a version that satisfies the requirement torch==1.4.0

具体错误原因也没必要细究，解决方法换一下安装命令即可：
pip install torch===1.4.0 torchvision===0.5.0 -f https://download.pytorch.org/whl/torch_stable.html
在运行完官方脚本的所有配置命令之后，输入以下命令运行：

python demo.py --vid_file sample_video.mp4 --output_folder output/ --display

不出意外的话会遇到一万个库缺失的报错，一个一个pip安装即可，接下来俺在导入pyopengl时遇到了以下错误：

  File "C:\Users\41656\Anaconda3\envs\pyVIBE\lib\site-packages\OpenGL\platform\egl.py", line 73, in EGL
raise ImportError("Unable to load EGL library", *err.args)
ImportError: ('Unable to load EGL library', 22, '找不到指定的模块。', None, 126, None, 'EGL', None)

查遍资料后也没解决这个问题，在github上提了issue，作者回复说可能是pyrender的问题，并表示他们没在win上运行过。

到这里就结束了win上的尝试，转向Ubuntu

Ubuntu踩坑

  1. 装 pytorch时 在跑完下载进度条就显示 进程被killed，解决方法：把虚拟机分配的内存改为8G

  2. tqdm报错，

    RuntimeError:cannot join current thread

    是老版本tqdm的bug，在4.31.1后就已经修复了这个bug，安装最新版tqdm即可

  3. DataLoader报错：

    RuntimeError: DataLoader worker (pid 4654) is killed by signal: Killed

    观察发现在加载数据时内存耗尽了，还是虚拟机内存不够的问题，batch改小即可
