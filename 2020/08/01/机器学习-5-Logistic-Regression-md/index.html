Discriminative v.s. Generative

Logistic Regression 的方法，我们把它称之为 discriminative 的方法；而我们用 Gaussian 来描述 posterior Probability 这件事，我们称之为 Generative 的方法

same model but different currency

在 Logistic Regression 里面，我们没有做任何实质性的假设，没有对 Probability distribution 有任何的描述，我们就是单纯地去找 b 和 w(推导过程中的假设只是便于理解和计算，对实际结果没有影响)

而在 Generative model 里面，我们对 Probability distribution 是有实质性的假设的，之前我们假设的是 Gaussian(高斯分布)，甚至假设在相互独立的前提下是否可以是 naive bayes(朴素贝叶斯)，根据这些假设我们才找到最终的 b 和 w

通常来说，Discriminative 的方法会比 Generative 的方法表现得更好

DM 模型和 GM 模型的优劣势

两个模型最大的区别在于，Generative 的 model 由人为做了某些假设，假设 data 来自于某个概率模型；而 Discriminative 的 model 是完全不作任何假设的。

因为 Generative model 人为地给数据加上了我们脑补的属性，这些属性会限制模型在某些方面的表现，所以有时 GM 模型的表现会出现问题。

但当我们的数据量 data 很少时或者数据有问题时，通过对 Generative model 做一些脑补和假设，反而可以把 data 的不足或是有问题部分的影响给降到最低，因为 DM 模型是完全依赖于数据的，数据量少时，DM 模型无法获得足够的信息，所以在这种情况下，GM 模型的表现会优于 DM 模型。
