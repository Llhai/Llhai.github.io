Spring面试笔记

Java

发布于：May 5, 2022

1. 介绍一下对 Spring IOC 的了解

  • IOC(Inversion of control)控制反转，它描述的是 Java 开发中对象创建以及管理的问题，它是一种设计思想，即将原本在程序中手动创建对象的控制权，交给框架来管理
  • 如，现有类 A 依赖于类 B：
    • 传统的开发方法，需要在类 A 中通过 new 关键字手动 new 一个 B 的对象出来
    • 使用 IOC 思想的开发方法，不需要通过 new 关键字来创建对象，而是通过 IOC 容器来帮助实例化对象，我们需要哪个对象，直接从 IOC 容器中取
  • IOC 降低了对象之间的耦合程度、使得资源管理变得容易

2. 介绍一下对 Spring AOP 的了解

  • AOP(Aspect-Oriented Programming) 面向切面编程，它能将那些与业务无关，但却为业务模块所共同调用的函数或者功能(如事务管理、日志管理、权限控制)封装起来
  • AOP 能减少系统的重复代码、降低模块间的耦合度、提高代码的可拓展性和可维护性

3. Spring Bean 相关内容

3.1 什么是 Spring Bean

IOC 容器中管理的对象，就是 Bean

3.2 将类声明为 Bean 的注解有哪些

  • @Component: 通用的注解，可以标注任意类
  • @Repository: 对应持久层即 Dao 层，主要用于数据库相关操作
  • @Service: 对应服务层，主要处理业务逻辑
  • @Controller：对应 Spring MVC 控制层，主要进行前端请求的处理、转发、重定向等

要点：

  1. @Service, @Controller, @Repository 是@Component 的拓展，从源码可以上来看，他们都派生自@Component 注解
  2. @Component、@Repository、@Service 默认是单例，而@Controller 默认多例

3.3 @Repository、@Controller、@Service 相对于@Component 的区别

首先，因为它们派生自@Component，所以对于 BeanDefinition 的自动检测和依赖注入，所有这些注解都是相同的，可以使用一个注解来替换另一个注解

  1. @Repository
    他提供了 PersistenceExceptionTranslationPostProcessor，将其添加到 Spring application context 后，可以捕获特定于平台的异常，并将它们添加到 Spring 的未检查异常重新抛出
  2. @Controller
    @Controller 是最特殊的注解，不能将此注解与@Service 或@Repository 等任何其它注解进行切换。因为调度器扫描用@Controller 注解的类，并检测其中用@RequestMapping 注解的方法，只能使用了@Controller 的类，才可以使用@RequestMapping 方法
  3. @Service
    @Service 目前没有特殊内容，它唯一的作用就是表示此类是用来处理业务逻辑

3.4 @Component 和@Bean 的区别是什么

  1. @Component 用于类，@Bean 用于方法
  2. @Component 是通过类路径扫描来自动侦测以及装配到 Spring 容器中；@Bean 注解则是在标有该注解的方法中定义产生这个 bean，并交由容器管理
  3. @Bean 注解比@Component 注解的自定义性更强，如当引用第三方库中的类需要装配到 Spring 容器或者下面代码中的例子，只能通过@Bean 实现

3.5 依赖注入的方法有哪些

  1. Field 注入

通过在属性上添加注解的方式注入

特点：

  • 代码简洁、美观
  • 新增字段时，不需要做过多修改
  • Spring 官方不推荐使用这种方法
  1. set 方法注入
  1. final + 构造函数 注入

特点：

  • 会使得代码臃肿，Field 字段过多时，使构造方法笨重
  • 新增 Field 时修改麻烦
  • 官方推荐使用这种方式注入

官方为什么推荐使用构造函数注入：

  1. 依赖不可变：使用 final 来约束 Field
  2. 依赖不可为空：在实例化时会检查构造函数参数是否为空，为空时抛出异常
  3. 单一职责：实例化时检查构造函数参数是否为空
  4. 避免循环依赖
  5. 避免空指针异常
  6. 更利于单元测试
  7. 保证返回客户端的代码的时候是完全初始化的状态
