单例模式的最优实现——枚举类

Java

发布于：May 12, 2022

什么是单例

单例模式的定义

单例模式就是对象在程序运行过程中只实例化一次，类似 Java 静态变量的思想，创建一个全局唯一对象。

单例实现思路

  • 将实例对象静态化
  • 将构造方法私有化，禁止通过构造方法创建实例
  • 提供一个 public 的静态方法，用来返回唯一实例

单例的优势

  • 只有一个对象，避免资源浪费问题
  • 在系统设置全局访问点，利于优化和共享资源访问

开发工具类库中的很多工具类都应用了单例模式，如线程池、缓存、日志，他们只需要创建一个对象，如果创建多份实例，可能会引发处理结果不一致、资源浪费等问题。

单例模式的几种实现

饿汉模式

在定义静态属性时，直接实例化单例对象

优点：

  • 这种方式简单粗暴，在类加载时就完成了对象的初始化，保证了线程安全

缺点；

  • 对象的初始化是在加载时完成的，即使没有使用这个类，它仍然已经被加载到了内存空间，造成资源浪费

懒汉模式

为了解决饿汉模式资源浪费的问题，懒汉模式才使用仅在使用这个实例时才创建实例

静态内部类单例模式

静态内部类是一种优秀的单例模式实现方法，它基于饿汉式的原理，同时解决了饿汉式的空间浪费问题。

这种方式的唯一实例由内部类创建，因为 JVM 在加载外部类的过程中，是不会加载静态内部类的，只有内部类的属性 or 方法被调用时，才会被加载并初始化。

这种方式是开源项目中常用的单例模式，不需要加锁便可以保证多线程安全，并且没有任何性能影响和空间浪费。

枚举类单例模式

枚举类单例模式是 Effective Java 作者推荐的单例模式，因为枚举类是线程安全的，并且指挥加载一次，这种特性非常适合于实现单例模式。

其他单例实现方法是可以通过反射或者序列化方法来破坏的，而枚举类单例不会
优点：

  • 不会被通过反射方式来破坏，枚举类通过反射构造对象时会抛出异常
  • 其他方式每次反序列化一个序列化的实例时，都会创建一个新的实例，而枚举类在序列化前后得到的实例是相同的
