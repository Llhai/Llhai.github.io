线程池

Java

发布于：Apr 20, 2022

1. 概念和优势

线程池是池化思想在线程管理方面的应用，使用线程池有以下三个优势：

  1. 降低资源消耗，重复利用已创建的线程，以降低线程创建和销毁造成的消耗
  2. 提高相应速度，当任务到达时，可以直接分配线程来运行，不需要等待线程创建
  3. 提供对线程的管理，可以进行线程的监控和调优

2. 七个核心参数

  1. corePoolSize：核心线程数
  2. maximumPoolSize：最大线程数
  3. keepAliveTime：非核心线程空闲时的存活时间大小
  4. unit：线程空闲存活时间单位
  5. workQueue：存放任务的等待队列
  6. threadFactory：创建线程的工厂，可以给创建线程设置名字，便于问题排查
  7. handle：饱和策略

3. 运行机制

4. 异常处理

  1. 使用 try-catch 在线程运行时捕获异常
  1. 使用 submit 提交任务，使用 Future.get 获取异常
  1. 为工作线程设置 UncaughtExceptionHandler，在 uncaughtException 方法中处理异常
  2. 重写 ThreadPoolExecutor 的 afterExecute 方法，处理传递的异常引用

5. 四种饱和策略

  1. AbortPolicy : 默认的饱和策略，抛出异常
  2. DiscardPolicy : 丢弃任务
  3. DiscardOldestPolicy : 丢弃队列里最老的任务，将当前这个任务提交给线程池
  4. CallerRunsPolicy : 交给线程池调用所在的线程进行处理

6. 阻塞队列类型

  1. ArrayBlockingQueue
    基于数组的有界队列，需要指定队列大小，按照 FIFO 排序
  2. LinkedBlockingQueue
    基于链表的队列，如果不设置容量，容量默认为 Integer.MAX_VALUE，此时为无界队列，吞吐量通常要高于 ArrayBlockingQuene；newFixedThreadPool 线程池使用了这个队列
  3. DelayQueue
    任务定时周期的延迟执行的队列，根据指定的执行时间从小到大排序，否则根据插入到队列的先后排序。newScheduledThreadPool 线程池使用了这个队列。
  4. PriorityBlockingQueue
    PriorityBlockingQueue（优先级队列）是具有优先级的无界阻塞队列，可以自定义比较器。
  5. SynchronousQueue
    一个不存储元素(内部容量为 0)的阻塞队列，每个插入操作必须等到另一个线程调用移除操作，否则插入操作一直处于阻塞状态，吞吐量通常要高于 LinkedBlockingQuene，newCachedThreadPool 线程池使用了这个队列。

7. 几种常见线程池

newFixedThreadPool

源码

参数：

  1. 核心线程数和最大线程数相同
  2. 阻塞队列为 LinkedBlockingQueue，导致最大线程数、饱和策略、存活时间参数失效

特点：

  1. 可以无限制的添加任务到队列，可能会造成 OOM
  2. 适合执行长期的任务，确保 CPU 在长期被工作线程使用的情况下，尽可能的少的分配线程

newCachedThreadPool

源码

参数：

  1. 核心线程数为 0，最大线程数为 Integer.MAX_VALUE
  2. 阻塞队列为 SynchronousQueue
  3. 非核心线程空闲存活时间为 60 秒

特点：

  1. 当提交任务的速度大于处理任务速度时，每次提交一个新任务，都会创建一个新线程。
  2. 适合执行任务量大，任务执行时间短的场景

8. 线程池状态

线程池有以下几种状态：

  1. RUNNING : 该状态的线程池会接收新任务，并且会处理阻塞队列中的任务;
  2. SHUTDOWN ：该状态的线程池不会接收新任务，但会处理阻塞队列中的任务；
  3. STOP ：该状态的线程不会接收新任务，也不会处理阻塞队列中的任务，而且会中断正在运行的任务；
  4. TIDYING ： 表明所有的任务已经运行终止，记录的任务数量为 0。
  5. TERMINATED: 表示该线程池彻底终止

9. 为什么《阿里巴巴 Java 开发手册》上要禁止使用 Executors 来创建线程池

从第 7 节可以看出，使用 Executors 创建出来的线程池使用的全都是无界队列，无界队列可以无限制地接受并保存任务，很可能造成 OOM 异常，并且会造成 maxinumPoolSize、keepAliveTime、handler 等参数失效
