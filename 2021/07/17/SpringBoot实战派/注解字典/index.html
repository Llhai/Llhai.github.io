Springboot注解

用在类名上的注解

  • @Controller : 标注控制器层，用于构建MVC模式的程序，在MVC中代表C
  • @RestController : 作用相当于@ResponseBody + @Controller
  • @Service : 声明一个业务处理类，用于标注服务层，处理业务逻辑
  • @Repository : 标注数据访问层
  • @Component : 用于把普通POJO(Plain Ordinary Java Objects)实例化到Spring容器中。当类不属于注解@Controller和@Service等时，就可以使用@Component来标注这个类。

Spring会把注解@Controller、@Service、@Repository、@Component标注的类纳入到Spring容器中进行管理

  • @Configuration : 用于标注配置类，可以由Spring容器自动处理，作为Bean的载体，用来指示一个类声明或多个@Bean方法
  • @Resource : 用于装配Bean
  • @Autowired : 表示被修饰的类需要注入对象。Spring会扫描所有被@Autowired标注的类，然后根据类型在IoC容器中找到匹配的类进行注入。
  • @RequestMapping : 用于处理地址映射，当用在类上时，表示类上所有相应请求的方法都以该地址作为父路径
  • @Transactional : 表明该类的所有public方法都具有该类型的事务属性。
  • @Qualifier : 用于标注哪一个实现类是需要注入的，它的参数名称为被注入的类中注解@Service标注的名称，常和@Autowired搭配使用

用在方法上的注解

  • @ResponseBody : 用在函数上，将函数返回值作为请求返回值，通常用来返回JSON数据或者是XML
  • @RequestBody : 用在方法参数前，将Json/XML格式的http消息转化为指定类型参数
  • @PathVariable : 用在方法参数前，将URL获取的参数映射到方法参数上
  • @Bean : 用在函数上，声明该方法的返回结果是一个由Spring容器管理的Bean，交给Spring管理，在MVC模型中对应着M

其他注解

  • @SpringBootApplication : 标注Spring Boot项目入口类
  • @EnableAutoConfiguration : 自动配置
  • @EnableScheduling : 用于开启计划任务
  • @EnableAsync : 用于开启异步注解功能
  • @ComponentScan : 用来扫描组件，可自动发现和装配一些Bean
  • @ASpec : 标注切面，用来配置事务、日志、权限验证

  • @ExceptionHandler : 用在方法上，表示遇到这个异常就执行该方法
  • @Value : 用在属性上，用于获取配置文件中的值
